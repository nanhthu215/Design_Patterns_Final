1️⃣  CRITICAL: Shared Shopping Cart Between Users
════════════════════════════════════════════════════════════════════════════

PROBLEM:
└─ CartContext used generic storage key: "cart-items"
└─ ALL users shared same localStorage key
└─ User A adds items → User B logs in → sees User A's items
└─ Users could purchase each other's items!

RISK LEVEL: 🔴 CRITICAL (Privacy violation, fraud risk)

FILES AFFECTED:
├─ frontend/src/contexts/CartContext.jsx
├─ frontend/src/services/CartService.js
└─ frontend/src/hooks/useCartSingleton.js

---

2️⃣  CRITICAL: Shared Order History Between Users
════════════════════════════════════════════════════════════════════════════

PROBLEM:
└─ OrderController.getAll() allowed fetching ALL orders
└─ No email filter enforcement
└─ Frontend might not pass email → returns everyone's orders
└─ Admin viewing page shows all customers' orders without permission

RISK LEVEL: 🔴 CRITICAL (Privacy violation, data breach)

FILES AFFECTED:
├─ backend/controllers/OrderController.js
└─ frontend/src/pages/Orders/OrderHistory.jsx

FIX #1: CartContext - User-Specific Storage Keys
────────────────────────────────────────────────

📍 FILE: frontend/src/contexts/CartContext.jsx

CHANGES:
✅ Helper function: getStorageKey(userEmail)
   └─ Old: "cart-items" (shared)
   └─ New: "cart-items-{email}" (user-specific)

✅ Import AuthContext to get current user email
   ```javascript
   const auth = useAuth();
   const currentUserEmail = auth?.user?.email || ...;
   ```

✅ Load cart when user changes (login/logout)
   ```javascript
   useEffect(() => {
     const raw = localStorage.getItem(storageKey); // ← user-specific key
     setItems(raw ? JSON.parse(raw) : []);
   }, [currentUserEmail, storageKey]); // ← reload on user change
   ```

✅ Update storage listener to use user-specific key
   ```javascript
   if (event.key === storageKey && event.newValue) { // ← now user-specific
   ```

RESULT:
✅ Each user has isolated cart
✅ Login = load that user's cart automatically
✅ Logout = clear cart (or load anonymous cart)
✅ No cart mixing between users

FIX #2: CartService Singleton - User-Specific Storage
──────────────────────────────────────────────────────

📍 FILE: frontend/src/services/CartService.js

CHANGES:
✅ Add static #currentUserEmail property
   ```javascript
   static #currentUserEmail = null;
   ```

✅ Add static #getStorageKey(email) method
   ```javascript
   static #getStorageKey(email) {
     if (!email) return 'cart-items-singleton-anonymous';
     return `cart-items-singleton-${email.toLowerCase()}`;
   }
   ```

✅ Add setCurrentUser(email) method
   ```javascript
   static setCurrentUser(email) {
     if (CartService.#currentUserEmail !== email) {
       CartService.#currentUserEmail = email;
       CartService.#loadFromStorage(); // ← reload cart for new user
     }
   }
   ```

✅ Update #loadFromStorage to use dynamic key
✅ Update #saveToStorage to use dynamic key
✅ Update #setupStorageListener to use dynamic key

RESULT:
✅ Singleton respects user context
✅ Automatic cart switch on user change
✅ Each user's cart isolated in localStorage
